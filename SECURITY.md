# Security Policy

## Supported Scope

Security reports are accepted for actively used code under:
- `30_Projects/*/prototype/`
- CI workflow definitions under `.github/workflows/`

## Reporting a Vulnerability

Please do not open public issues for sensitive vulnerabilities.

Preferred process:
1. Use GitHub Security Advisories (private report) for this repository.
2. Include reproduction steps, affected paths, impact assessment, and suggested mitigation.
3. If advisory reporting is unavailable, open an issue with minimal details and request a private channel.

## Response Expectations

- Initial triage target: within 5 business days.
- Confirmed issues are prioritized by severity and exploitability.
- Fixes may be released with minimal disclosure until mitigation is available.

## Safe Disclosure

- Avoid publishing exploit details before a fix is merged and released.
- Coordinate disclosure timing with maintainers.
